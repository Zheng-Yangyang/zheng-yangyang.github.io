2.git的基本使用方法

2.1 git init

用于在目录中创建新的Git仓库，可以用 git init, 然后就会自动初始化，并生成一个.git文件夹

2.2 git status

查看当前工作区的状态，若有文件未加到暂存区里面，会提示你用git add命令把他加入暂存区

2.3 git add

git add是把工作区的文件添加到暂存区，可以单独添加文件，使用git add +文件名，也可以添加所有文件 git add .

添加之后再查看状态就是如下所示了

2.4 git rm –catched

我们可以看上图的命令提示，提示说可以使用git rm –cached 来恢复文件到工作区，我们使用一下，发现又回到了原来没加入暂存区的状态了

2.5 git commit

git commit 可以吧暂存区的内容发布到本地仓库，便于后期的版本回溯，git commit 如果不加入任何参数，就会默认进入到nano的编辑模式，一种类似于vim的工作状态，在这里你可以给你的commit增加备注消息，更常见的方法是，直接使用 git commit -m “你的备注”
